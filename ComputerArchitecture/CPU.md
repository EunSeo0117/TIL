# CPU와 명령어 처리

## 면접대비 질문

### CPU의 구조와 주요 구성 요소(ALU, 제어 유닛, 레지스터)에 대해 설명해주세요.
CPU는 프로그램의 명령어를 해석하고 실행하는 컴퓨터 핵심장치이다.  
주요 구성요소는 ALU, 제어유닛, 레지스터다.  

ALU는 산술연산과 논리연산을 수행하는 장치앋. 덧셈,뺄셈, AND, OR같은 계산을 처리한다.  

제어유닛은 명령어를 해석하고 각 하드웨어가 어떤 동작을 해야하는지 제어 신호를 보내는 역할을 한다.  

레지스터는 CPU내부에 있는 매우 빠른 메모리로, 현재 실행중인 명령어나 연산에 필요한 데이터를 일시적으로 저장한다. 일반 메모리보다 훨신 빠르기때문에 CPU는 가능한 한 레지스터를 활용해 연산을 수행한다.

### 명령어 사이클(Instruction Cycle)의 단계(Fetch, Decode, Execute)에 대해 설명해주세요.
CPU는 프로그램을 실행할때 명령어 사이클이라는 반복과정을 수행한다.  
이 과정은 일반적으로 Fetch, Decode, Execute단계로 구성된다.  

Fetch단계에서는 메모리에서 다음에 실행할 명령어를 가져온다. 이때 프로그램 카운터(PC)가 가리키는 주소의 명령어를 읽어온다.  

Decode단계에서는 가져온 명령어를 해석하여 어떤연산을 수행해야 하는지 판단한다. 예를들어 산술연산인지, 메모리 접근인지 등을 분석한다.  

Execute단계는 실제 연산을 수행한다. ALU연산을 수행하거나, 메모리 접근을 하거나, 레지스터 값을 변경하는 등의 작업이 이뤄진다. 

### 파이프라이닝(Pipelining)이 무엇이고, 어떻게 CPU 성능을 향상시키나요?
파이프라이닝은 명령어 실행 과정을 여러 단계로 나누고, 서로 다른 명령어를 동시에 처리하는 기술이다.   
예를들어 fetch, decode, execute단계가 있다면, 한 명령어가 decode단계에 있을때 다음 명령어는 fetch단계에 들어갈수있다.  
이 방식은 공장 생산라인과 유사하다.  
여러 명령어들을 동시에 처리할수있기때문에, cpu처리량(throughtput)이 증가하고 결과적으로 전체 성능이 향상된다.

### 파이프라인 해저드(Pipeline Hazard)의 종류와 해결 방법에 대해 설명해주세요.
파이프라인 해저드는 파이프라인 처리 과정에서 명령어의 실행이 지연되거나 충돌이 발생하는 상황을 의미한다.  
구조적 해저드는 동일한 하드웨어 자원을 여러 명령어가 동시에 사용하려할때 발생한다.  
이를 해결하기위해 자원을 추가하거나 명령어 실행을 지연시키는 방법을 사용한다.  
데이터 해저드는 이전 명령어의 결과가 아직 계산되지 않았는데, 다음명령어가 그 데이터를 사용하려고 할때 발생한다.  
이를 해결하기위해 포워딩이나 파이프라인 스톨을 사용한다.  
제어 해저드는 분기 명령어 떄문에 다음 실행할 명령어를 확저할수없는 경우 발생한다.  
이를 해결하기 위해 분기 예측이나 지연 분기같은 기법을 사용한다. 

### 분기 예측(Branch Prediction)이 무엇이고 왜 중요한가요?
분기 예측은 조건 분기 명령어에서 프로그램의 흐름이 어느 방향으로 갈지를 CPU가 미리 예측하는 기술이다.  
분기 명령어는 다음 실행할 명령어의 위치를 바꾸기 때문에 파이프라인이 멈출수있다.  
이때 CPU는 과거 실행 패턴을 기반으로 분기 결과를 예측하고 다음 명령어를 미리 가져온다.  
예측이 맞으면 파이프라인이 계속 유지되어 성능이 크게 향상된다.  
반대로 예측이 틀리면 파이프라인을 비우고 다시 실행해야하므로 성능이 감소한다.  
따라서 분기 예측 정확도는 CPU성능에 매우 중요한 요소이다. 

### Out-of-Order Execution이 무엇이고, 어떤 이점이 있나요?
명령어가 프로그램에 작성된 순서대로가 아니라, 실행가능한 명령어부터 먼저 실행하는 기술이다.  
일부 명령어는 이전 명령어의 결과를 기다려야하지만, 다른 명령어는 의존성이 없다면 먼저 실행할수있다.  
CPU는 이러한 명령어를 먼저 실행하여 연산 자원을 효율적으로 활용한다.  
이 방식의 장점은 CPU의 유휴시간을 줄이고 전체 처리 성능을 향상시킬수있다는것이다. 

### CISC와 RISC 아키텍처의 차이점에 대해 설명해주세요.
CISC는 복잡한 명령어를 제공하는 구조이다.  
하나의 명령어로 여러 작업을 수행할수있으며, 명령어 길이와 형태가 다양한 특징이있다. 대표적인 예시는 x86아키텍쳐이다.  
RISC는 단순하고 짧은 명령어 집합을 사용하는 구조이다.  
대부분의 명령어가 한 클럭에 실행되도록 설계되어 있으며 명령어 형식이 단순하다. ARM아키텍쳐가 대표적인 예시이다.  
CISC는 프로그램 코드 길이를 줄이는데 유리하고, RISC는 단순한 구조 덕분에 고속 파이프라이닝과 효율적인 하드웨어 설계에 유리하다.

### x86과 ARM 아키텍처의 차이점은 무엇인가요?
x86은 인텔과 Amd에서 사용하는 cisc기반 아키텍쳐로 복잡한 명령어 집합을 사용한다.  
높은 성능과 호환성을 중심으로 발전해 왔으며 주로 데스크톱과 서버 환경에서 사용된다.  
ARM은 RISC기반 아키텍쳐로 단순한 명령어 구조를 가지고있으며, 전력 효율이 높은것이 특징이다.  
그래서 모바일 기기, 임베디드, Iot기기에서 많이 사용된다.  
x86은 성능중심의 cisc아키첵쳐, Arm은 전력효율 중심의 Risc아치택쳐라는 차이가있다.
