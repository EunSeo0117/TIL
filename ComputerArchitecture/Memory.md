# Memory(메모리)

# 면접대비 질문

## 메모리 계층 구조

### 메모리 계층 구조(Memory Hierarchy)에 대해 설명해주세요.
메모리 계층 구조는 cpu와 저장장치 사이의 성능 차이를 줄이기위해 여러 단계의 메모리를 계층적으로 구성한 구조를 의미한다.  
cpu에 가까울수록 속도는 빠르지만, 용량이 작고 가격이 비싸며, cpu에서 멀어질수록 속도는 느리지만 용량이 크고 가격이 저렴하다.  
구조는 1.레지스터 2.캐시(레이어1,2,3), 메인메모리(RAM), 세컨드스토리지(SSD, HDD)가 있다.  
이 구조를 통해 자주 사용하는 데이터를 빠른 메모리에 두어 전체 시스템 성능을 향상시킬수있다.

### 레지스터, 캐시, 메인 메모리, 보조 기억장치의 속도와 용량을 비교해주세요.
| 메모리                         | 속도    | 용량    | 위치           |
| --------------------------- | ----- | ----- | ------------ |
| Register                    | 가장 빠름 | 매우 작음 | CPU 내부       |
| Cache                       | 매우 빠름 | 작음    | CPU 내부 또는 근접 |
| Main Memory (RAM)           | 보통    | 수 GB  | 메인 메모리       |
| Secondary Storage (SSD/HDD) | 느림    | 매우 큼  | 디스크          |

속도는 레지스터 -> 캐시 -> 램 -> 디스크 순으로 느려지고, 용량은 반대로 증가한다.

#### 캐시 메모리가 왜 필요하고, 어떤 원리로 동작하나요?
cpu처리 속도는 매우 빠르지만, 메인메모리(ram)은 상대적으로 느리다.  
그래서 cpu가 매번 램에 접근하면 성능이 크게 저하된다.  
이를 해결하려고 자주사용하는 데이터를 캐시에 저장하여 cpu가 빠르게 접근하도록 하는것이 캐시 메모리 역할이다.  
캐시는 프로그램이 특정 데이터를 반복적으로 접근하는 특성을 이용해서 동작한다.

캐시는 프로그램의 메모리 접근이 특정 패턴을 가지는 지역성(Locality) 원리를 이용하여 동작한다.
- 시간 지역성 : 최근 접근 데이터는 다시 접근될 가능성이 높다
- 공간 지역성 : 특정 주소 접근 시 인접 주소도 접근될 가능성이 높다

### 캐시의 지역성(Locality) 원리에 대해 설명해주세요.
캐시는 프로그램이 특정 데이터나 메모리 영역을 반복적으로 사용하는 특성을 이용한다.  
이를 지역성이라고 하고 두가지 종류가 있다.  
1. 시간 지역성 
2. 공간지역성

### 시간 지역성(Temporal Locality)과 공간 지역성(Spatial Locality)의 차이는 무엇인가요?
시간 지역성은 최근에 접근한 데이터가 가까운 미래에도 다시 사용될가능성이 높다는 특성을 의미한다.  
반복문에서 같은 변수를 계속사용하는 경우가 해당된다.  
공간지역성은 어떤데이터에 접근하면 그 주변 메모리 주소의 데이터도 곧 접근될 가능성이 높다는 특성이다.  
배열을 순차적으로 탐색하는 경우다.

### 캐시 라인(Cache Line)이 무엇이고, 크기가 성능에 어떤 영향을 미치나요?
캐시 라인은 캐시가 메모리에서 데이터를 가져올때 사용하는 최소단위이다.  
cpu가 하나의 데이터를 요청하면 실제로는 해당 데이터가 포함된 일정 크기의 블록을 메모리에서 가져온다.  
캐시라인의 크기가 크면 공간지역성을 활용하기 좋지만, 필요없는 데이터도 함께 가져올수있다.  
반대로 너무 작으면 메모리 접근 횟수가 증가한다.  
캐시라인 크기는 성능과 효율성 사이의 균형을 고려해 설계된다. 

### 캐시 히트(Cache Hit)와 캐시 미스(Cache Miss)에 대해 설명해주세요.
캐시 히트는 cpu가 요청한 데이터가 캐시에 존재하는 경우다.  
이 경우 매우 빠르게 데이터를 읽을수있다.  
캐시미스는 cpu가 요청한 데이터가 캐시에 없는 경우다.  
이 경우 메인 메모리에서 데이터를, 가져와서 캐시에 저장한후 cpu에 전달된다.
캐시미스가 많아질수록 성능이 크게 저하된다.  

### 캐시 매핑 방식(Direct Mapped, Fully Associative, Set Associative)의 차이점을 설명해주세요.
| 방식                | 특징                                 | 장점              | 단점                    |
| ----------------- | ---------------------------------- | --------------- | --------------------- |
| Direct Mapped     | 메모리 블록이 캐시의 **한 위치에만 저장 가능**       | 구조 단순, 접근 속도 빠름 | 충돌(Cache Conflict) 많음 |
| Fully Associative | 메모리 블록이 캐시의 **어느 위치든 저장 가능**       | 충돌 최소           | 하드웨어 복잡, 비용 증가        |
| Set Associative   | 캐시를 여러 **Set으로 나누고 해당 Set 내에서 저장** | 충돌 감소, 성능 균형    | Direct보다 구조 복잡        |

캐시 매핑 방식은 메인 메모리의 데이터를 캐시에 어떤 위치에 저장할지 결정하는 방식이다.

Direct Mapped Cache  
메모리의 특정 블록이 캐시의 특정 위치에만 저장될수있는 방식이다.  
구조가 단순하고, 접근속도가 빠르지만, 서로 다른 메모리 블록이 같은 캐시 위치에 매핑되면 충돌이 발생하여 캐시 미스가 증가하는 단점이 있다.

Fully Associative Cache  
방식은 메모리 블록이 캐시의 어느 위치에나 저장될수있는 방식이다.  
충돌이 거의 발생하지 않지만, 모든 캐시 라인을 탐색해야하므로 하드웨어 비용과 연산 비용이 증가한다.  

set asspcivative cache  
direct 방식과 fully associative 방식의 절충안이다.  
캐시를 여러개의 set으로 나누고, 특정 메모리 블록은 해당 set내부의 여러 라인중 하나에 저장될수있다.  
충돌을 줄이면서도 비교 연산 비용을 낮출수있어 현대 cpu에서 가장 많이 사용되는 방식이다. 

### 캐시 교체 정책(LRU, LFU, FIFO 등)에 대해 설명해주세요.
| 정책            | 교체 기준             | 특징          |
| ------------- | ----------------- | ----------- |
| LRU           | 가장 오래 사용되지 않은 데이터 | 시간 지역성 활용   |
| LFU           | 접근 빈도 가장 낮은 데이터   | 빈도 기반       |
| FIFO          | 가장 먼저 들어온 데이터     | 구현 단순       |
| Random        | 무작위 선택            | 관리 비용 최소    |
| MRU           | 가장 최근 사용 데이터      | 순차 접근에 유리   |
| Second Chance | FIFO + 참조 비트      | FIFO 개선     |
| Pseudo-LRU    | LRU 근사 구현         | CPU 캐시에서 사용 |
| ARC           | LRU + LFU 조합      | 적응형 캐시      |


캐시 공간이 가득찼을때 어떤 데이터를 제거할지 결정하는 알고리즘이다.  

LRU(Least Recently Used)  
가장 오랫동안 사용되지 않은 데이터를 제거하는 방식이다.  
시간 지역성 원리를 잘 반영하기 때문에 실제 시스템에서 많이 사용된다.

LFU(least Frequently Used)  
가장 적게 사용된 데이터를 제거하는 방식이다.  
접근 횟수를 기준으로 하기 떄문에 특정 데이터가 장기간 사용되는 환경에 유리하다. 

FIFO(first in first out)  
가장먼저 캐시에 들어온 데이터를 제거하는 방식이다.  
구현이 간단하지만 실제 접근 패턴을 고려하지 않기때문에 효율이 떨어질수있다.  

### Write-Through와 Write-Back 캐시 쓰기 정책의 차이점은 무엇인가요?
| 구분      | Write-Through                   | Write-Back                          |
| ------- | ------------------------------- | ----------------------------------- |
| 동작 방식   | 캐시에 데이터를 쓸 때 **메인 메모리에도 즉시 기록** | 캐시에만 먼저 기록하고 **캐시에서 교체될 때 메모리에 반영** |
| 메모리 일관성 | 항상 메모리와 캐시 데이터가 동일              | 캐시와 메모리 데이터가 일시적으로 다를 수 있음          |
| 성능      | 메모리 접근이 많아 상대적으로 느림             | 메모리 접근이 줄어 성능이 더 좋음                 |
| 구현 복잡도  | 단순                              | Dirty bit 관리 등으로 복잡                 |
| 안정성     | 메모리 항상 최신 상태                    | 캐시 손실 시 데이터 유실 가능                   |


캐시쓰기 정책은 cpu가 데이터를 수정할때 캐시와 메인메모리를 어떻게 동기화할지 결정하는 방식이다. 

write-through  
캐시에 데이터를 쓰는 동시에 메인메모리에도 바로 기록하는 방식이다.  
데이터 일관성을 유지하기 쉽지만 메모리 쓰기 연산이 증가하여 성능이 떨어질수있다. 

write-back  
캐시에만 데이터를 먼저 기록하고, 캐시 블록이 교체될떄 메인메모리에 기록하는 방식이다.  
메모리 쓰기 횟수를 줄여 성능이 향상되지만, 캐시와 메모리 사이 데이터 일관성 관리가 필요하다.


### L1, L2, L3 캐시의 차이점과 각각의 역할에 대해 설명해주세요.
| 구분    | L1 Cache            | L2 Cache           | L3 Cache          |
| ----- | ------------------- | ------------------ | ----------------- |
| 위치    | CPU 코어 내부           | CPU 코어 내부 또는 코어 근처 | CPU 칩 내부          |
| 속도    | 가장 빠름               | L1보다 느림            | L2보다 느림           |
| 용량    | 가장 작음 (보통 32~128KB) | 중간 (256KB~1MB 수준)  | 가장 큼 (수 MB~수십 MB) |
| 공유 여부 | 코어별 전용              | 보통 코어별 전용          | 여러 코어가 공유         |

cpu캐시는 여러 단계로 구성되어있으며, cpu와 가까울수록 속도는 빠르지만 용량이 작다.  

l1는 cpu코어 내부에 존재하는 가장 빠른 캐시이다.  
용량은 보통 수십 kb수준이며, 명령어 캐시와 데이터 캐시로 분리되어 있는 경우가 많다.  
용량은 매우 작지만 접근 시간이 가장 짧아 CPU 성능에 큰 영향을 준다.

l2는 l1보다 크고 속도는 조금 느린 캐시이다.  
보통 각 cpu코어에 하나씩 존재하며 수백kb에서 수mb정도의 크기를 가진다.  
대부분 코어별 전용 캐시로 동작하여 해당 코어의 작업 데이터를 저장한다.

l3캐시는 여러 cpu코어가 공유하는 캐시이다.  
용량은 수 mb에서 수십 mb수준이며, l2보다 느리지만, 메인메모리보다 훨씬 빠르다.   
멀티코어 환경에서 코어 간 데이터 공유 비용을 줄이는 역할을 한다.

### 캐시 일관성(Cache Coherence) 문제와 해결 방법에 대해 설명해주세요.
멀티코어 시스템에서 여러 cpu코어가 각자의 캐시를 가지고 동일한 메모리 데이터를 사용할수있다.  
이때 한 코어가 데잍터를 수정하면 다른 코어의 캐시에 있는 같은 데이터가 오래된 값이 되는 문제가 발생하는데, 이를 캐시 일관성문제라고 한다.  

이를 해결하기위해 캐시 일관성 프로토콜이 사용된다.  
1. Write Invalidate 방식  
    한 코어가 데이터를 수정하면 다른 코어의 캐시에 있는 동일한 데이터 블록을 무효화(Invalid) 하는 방식이다.
    다른 코어가 해당 데이터를 다시 접근하면 메모리나 최신 캐시에서 데이터를 다시 가져오게 된다.
    불필요한 데이터 전파를 줄일 수 있어 가장 널리 사용되는 방식이다.
2. Write Update 방식  
   데이터가 수정되면 다른 캐시에 있는 동일한 데이터도 함께 업데이트하는 방식이다. 모든 캐시가 항상 동일한 값을 유지하지만, 수정할 때마다 모든 캐시에 데이터를 전달해야 하므로 버스 트래픽이 증가할 수 있다.
3. MESI 캐시 일관성 프로토콜
    | 상태        | 의미                  |
    | --------- | ------------------- |
    | Modified  | 캐시에만 수정된 최신 데이터가 존재 |
    | Exclusive | 캐시에만 존재하지만 메모리와 동일  |
    | Shared    | 여러 캐시에 동일 데이터 존재    |
    | Invalid   | 데이터가 유효하지 않음        |
    
    이 프로토콜을 통해 한 코어가 데이터를 수정하면 다른 캐시의 해당 데이터는 Invalid 상태가 되어 최신 데이터를 다시 가져오도록 하여 일관성을 유지한다.

## 메모리 관리

### 가상 메모리(Virtual Memory)가 무엇이고 왜 필요한가요?
프로세스가 실제 물리 메모리보다 더 큰 메모리를 사용할수있도록 하는 메모리관리 기법이다.  
운영체제는 프로그램이 사용하는 주소를 가상 주소 공간으로 제공하고, 실제 데이터는 필요할때만 물리 메모리에 적재한다.  
이 방식은 프로그램마다 독립적인 메모리 공간을 제공하여 프로세스간 메모리 보호를 가능하게하고, 물리메모리보다 큰 프로그램도 실행할수있도록 하며, 메모리 사용효율을 높이는 장점이있다. 

### 물리 주소(Physical Address)와 논리 주소(Logical Address)의 차이는 무엇인가요?
논리 주소는 프로그램이 사용하는 주소로, cpu가 생성하는 주체이다.  
이 주소는 실제 메모리 위치가 아닌 가상주소 공간에서의 위치를 의미한다.  

물리주소는 실제 ram에서의 주소이다.  
cpu가 생성하는 논리주소는 mmu를 통해 변환되어 물리 주소로 매핑된후 메모리에 접근하게된다. 

### 페이지 테이블(Page Table)의 역할과 구조에 대해 설명해주세요.
가상주소를 물리주소로 변환하기 위한 매핑 정보를 저장하는 자료구조이다.  
가상 메모리는 주소공간을 페이지 단위로 나누고, 물리메모리는 프레임 단위로 나눈다.  
페이지 테이블의 각 항목은 가상 페이지 번호와 물리프레임 번호의 매핑 정보를 가지고있으며, 추가적으로 유효비트, 수정비트, 접근비트 등의 상태 정보도 포함하다.  
이를통해 운영체제는 어떤 가상 페이지가 어떤 물리프레임에 위치하는지 알수있다. 

### TLB(Translation Lookaside Buffer)가 무엇이고 왜 필요한가요?
가상 주소와 물리주소의 변환 결과를 캐싱하는 고속 메모리이다.  
주소 변환 과정에서 매번 페이지 테이블을 조회하면 메모리 접근이 추가로 발생해 성능이 저하된다.  
이를 해결하기위해 최근에 사용된 주소변환 정보를 TLB에 저장해두고, 같은 수조 접근이 발생하면 페이지 테이블을 거치지 않고 빠르게 주소 변환을 수행할수있다. 

### 페이지 폴트(Page Fault)가 발생했을 때 처리 과정을 설명해주세요.
페이지폴트는 프로세스가 접근하려는 페이지가 현재 물리메모리에 존재하지 않을때 발생하는 예외상황이다. 

처리과정
1. cpu가 가상주소 접근시 페이지 테이블 확인
2. 해당 페이지가 메모리에 없으면 페이지 폴트 인터럽트 발생
3. 운영체제가 디스크의 스왑 영역에서 필요 페이지 찾음
4. 물리 메모리에 빈 프레임이 있으면 해당 페이지를 로드
5. 빈프레임 없으면 페이지 교체 알고리즘으로 기존 페이지 제거
6. 페이지 테이블을 업데이트한후 프로그램 실행 재개

### 메모리 단편화(Fragmentation)의 종류와 해결 방법은 무엇인가요?
메모리 단편화는 메모리를 할당하고, 해제하는 과정에서 사용하지 못하는 작은 공간이 발생하는 현상이다.   

외부 단편화는 메모리 사이사이에 작은 빈공간이 생겨 연속적인 메모리 할당이 어려워지는 현상이다.  
해결법은 메모리 압축이나 페이징 방식이 있다.  

내부 단편화는 할당된 메모리 블록 내부에 사용되지 않는 공간이 남는 현상이다.  
이는 고정 크기 블록을 사용할때 발생하며, 해결 방법으로 블록크기 조절이나 세분화된 메모리 관리방식이 있다. 


### DMA(Direct Memory Access)가 무엇이고 어떤 장점이 있나요?
dma는 cpu를 거치지 않고 i/o 장치가 메모리에 직접 접근하여 데이터를 전송하는 방식이다.  
일반적 i/o에서는 cpu가 데이터를 직접 읽고 써야하지만, DMA를 사용하면 DMA컨트롤러가 메모리와 장치사이의 데이터 전송을 담당한다.  
이를 통해 cpu사용률을 줄이고 대량 데이터 전송 성능을 향상시킬수있다.


### 메모리 버스(Memory Bus)와 대역폭(Bandwidth)에 대해 설명해주세요.
메모리 버스는 cpu와 메모리 사이에서 데이터, 주소, 제어신호를 전달하는 통로이다.  
cpu가 메모리에 접근할때 이 버스를 통해 데이터가 이동한다.  
메모리 대역폭은 단위 시간 동안 메모리 버스를 통해 전송할수있는 데이터양을 의미한다.  
일반적으로 버스폭(비트수)과 클럭속도에 의해 결정되며, 대역폭이 높을수록 cpu와 메모리간 데이터 전송 속도가 빨라져 전체 시스템 성능이 향상된다.
