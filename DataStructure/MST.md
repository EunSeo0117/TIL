# MST(Minimum Spanning Tree)

## 개념
MST는 가중치가 있는 무방향 그래프에서 모든 정점을 연결하면서 간선의 가중치 합이 최소가 되는 트리이다.
트리는 사이클이 없고, 정점이 V개일때 간선이 V-1개인 그래프이므로 MST도 이 조건을 만족한다. 

## 특징
MST는 모든 정점을 연결해야하며 사이클이 존재하지 않는다.  
또한 간선의 개수는 항상 v-1개이다.  
그래프가 연결되어 있지 않다면 MST는 존재하지 않고, 이 경우 각 연결 요소마다 MST가 만들어진 Minimum Spanning Forest가 된다.  
간선 가중치가 같은 경우에는 여러개의 MST가 존재할수있다. 

## 시간복잡도
- 크루스칼 알고리즘
  - 간선을 가중치 기준으로 정렬한두 작은 간선부터 선택하며 사이클이생기지 않도록 Union-Find로 검사한다. 
  - 간선 정렬이 지배적이기 때문에 시간복잡도는 O(E log E)이다. 
- 프림 알고리즘
  - 하나의 정점에서 시작하여 현재 트리와 연결되는 최소 간선을 선택하며 확장한다.
  - 우선순위 큐를 사용하면 시간복잡도는 O(E log V)이다. 

## 면접 대비 질문

### MST가 무엇이고, 어떻게 구할 수 있을지 설명해 주세요.
MST는 가중치가 있는 무방향 그래프에서 모든 정점을 연결하면서 간선 가중치의 합이 최소가 되는 트리이다.  
대표적인 알고리즘은 크루스칼과 프림 알고리즘이 있다.  
크루스칼은 간선을 가중치 기준으로 정렬한뒤 사이클이 발생하지 않도록 간선을 선택해 나가는 방식이다.  
프림알고리즘은 하나의 정점에서 시작해 현재 트리와 연결되는 최소 간선을 선택하며 트리를 확장해나가는 방식이다.

### Kruskal 알고리즘에서 사용하는 Union-Find 자료구조에 대해 설명해 주세요.
유니온파인드는 서로소 집합을 관리하는 자료구조로, 여러 원소가 어떤 집합에 속하는지 빠르게 관리하기 위해 사용된다.  
주요연산은 특정원소가 속한 집합의 대표를 찾는 find연산과, 두 집합을 하나로 합치는 Union연산이 있다.  
크루스칼 알고리즘에서는 간선을 선택할때 두 정점이 이미 같은 집합 속에 있다면 사이클이 발생하기 때문에 해당 간선을 선택하지 않는다. 이를 판단하기 위해 유니온파인드를 사용한다. 

### Kruskal 과 Prim 중, 어떤 것이 더 빠를까요?
그래프 구조에따라 다르다.  
간선이 적은 희소 그래프에서는 간선을 정렬한뒤 처리하는 크루스칼이 효율적인 경우가 많다.  
반면 간선이 많은 밀집 그래프에서는 프림 알고리즘이 더 효율적으로 동작하는 경우가 많다.

### Kruskal 과 Prim 알고리즘을 통해 얻어진 결과물은 무조건 트리인가요? 만약 그렇다면 증명해 주세요. 그렇지 않다면, 반례를 설명해 주세요.
그래프가 연결되어 있다면 두 알고리즘의 결과는 항상 트리이다.  
두 알고리즘 모두 간선을 선택할때 사이클이 발생하지 않도록 보장하며, 모든 정점을 연결할 때가지 간선을 선택한다.  
이 과정에서 선택되는 간선의 수는정확히 V-1개가 되므로 결과 그래프는 사이클이 없는 연결그래프가 되고, 이는 트리의 정의를 만족한다.  
단, 그래프가 연결되어 있지 않다면 하나의 트리가 아니라 여러개의 트리로 이루어진 Minimum Spanning Forest가 된다.
