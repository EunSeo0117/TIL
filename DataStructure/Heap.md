# Heap

## 개념
완전이진트리 기반의 자료구조로, 부모노드와 자식노드 사이에 특정한 대소 관계를 만족하는 구조이다.  
MaxHeap: 부모 >= 자식,  
MinHeap: 부모 <= 자식 인 구조로 루트노드에 항상 최댓값 또는 최솟값이 위치한다. 

## 특징
- 완전 이진 트리 형태이다.
- 마지막 레벨을 제외하고는 모두 채워져있다.
- 마지막 레벨은 왼쪽부터 채워진다.
- 배열로 구현 가능하다.
  - 부모인덱스 : i
  - 왼쪽 자식: 2i + 1
  - 오른쪽 자식: 2i + 2
  - 부모: (i - 1) / 2

## 힙 종류
- MaxHeap
  - 부모노드가 자식노드보다 항상 크거나 같다.
  - 최댓값을 빠르게 꺼낼수있다.
- Min Heap
  - 부모노드가 자식 노드보다 항상 작거나 같다.
  - 최솟값을 빠르게 꺼낼수 있다.

## 시간복잡도
| 연산             | 시간복잡도    |
| -------------- | -------- |
| 삽입 (Insert)    | O(log N) |
| 삭제 (루트 삭제)     | O(log N) |
| 조회 (루트 확인)     | O(1)     |
| 힙 생성 (Heapify) | O(N)     |

## 동작 원리
- 삽입
  1. 트리 마지막 위치에 삽입
  2. 부모와 비교
  3. 조건이 맞지않으면 교환
  4. 루트까지 반복
- 삭제
  1. 루트 제거
  2. 마지막 노드를 루트로 이동
  3. 자식과 비교후 교환
  4. 조건 만족할 떄까지 반복

## 힙과 이진탐색 트리 차이
| 구분      | Heap     | BST           |
| ------- | -------- | ------------- |
| 구조      | 완전 이진 트리 | 이진 트리         |
| 정렬 상태   | 부분 정렬    | 전체 정렬         |
| 탐색      | O(N)     | O(log N) (평균) |
| 최댓값/최솟값 | O(1)     | O(log N)      |

## 활용 사례
- 우선순위 큐
- 다익스트라 알고리즘
- 힙 정렬
- 스케쥴링 시스템
- Top-K문제 해결

## 면접대비 질문

### 힙에 대해 설명해 주세요.
힙은 완전이진트리 기반의 자료구조로, 부모와 자식간의 대소 관계를 유지하는 구조이다.  
minheap은 부모보다 항상 자식이 크고, maxheap은 부모가 항상 자식보다 큰 조건을 만족한다.  
따라서 루트에는 항상 최솟값 또는 최댓값이 위치한다.  
힙은 전체 정렬이 아닌 부분정렬 구조이며, 우선순위 큐 구현에 주로 사용된다.

### 힙을 배열로 구현한다고 가정하면, 어떻게 값을 저장할 수 있을까요?
힙은 완전 이진 트리이기 떄문에 배열로 효율적으로 표현할수있다.  
레벨순서로 저장하면 인덱스 계산만으로 부모-자식 관계를 찾을수있다. 
부모인덱스를 (i-1)/2로, 왼쪽 자식은 2i+1, 오른쪽 자식은 2i+2로 구성하면 된다.  

### 힙의 삽입, 삭제 방식에 대해 설명하고, 왜 이진탐색트리와 달리 편향이 발생하지 않는지 설명해 주세요.
삽입은 배열의 마지막위치에 삽입한 후, 부모와 비교하여 조건 위반시 swap하며 루트까지 반복한다.(up-heap/bubble-up)  
삽입 시간복잡도는 logN 이다.  
삭제는 루트를 제거하고, 마지막 노드를 루트로 이동시킨후, 자식과 비교하여 조건위반시 swap하며 조건만족때까지 반복한다(down-heap/heapify)  
삭제 시간복잡도는 logN 이다.  
이진탐색 트리와 달리 편향이 발생하지 않는 이유는 완전이진트리 구조를 강제하기떄문이다.  
이진탐색트리는 삽입 순서에따라 한쪽으로 치우칠수있지만, 힙은 항상 마지막 레벨의 왼쪽부터 채우기때문에 구조가 무조건 균형에 가깝다.  
따라서 편향적인 구조가 발생하지않는다.

### 힙 정렬의 시간복잡도는 어떻게 되나요? Stable 한가요?
힙정렬의 시간복잡도는 heap구성은 o(n), n번 노드 삭제는 n*logN이다.  
전체 시간 복잡도는 NlogN이므로 최선/평균/최악 모두 동일하다.  
힙정렬은 스테이블 하지않다. 그이유는 루트와 마지막 원소를 swap하는 과정에서 동일한 값의 상대적 순서가 보장되지 않기때문이다.
