# 재귀 (Recursion)

## 개념
재귀는 함수가 자기 자신을 다시 호출하여 문제를 해결하는 방식이다.  
큰 문제를 동일한 구조의 더 작은 문제로 나누어 해결 할 수 있을때 주로 사용된다.  
재귀 함수는 반드시 재귀 호출과 종료 조건(Base Case) 을 포함해야 하며, 종료 조건이 없으면 무한 호출로 인해 Stack Overflow가 발생할 수 있다.

## 특징
- 문제를 작은 문제로 분할 하여 해결한다.
- 코드가 간결해지는 경우가 많다.
- Call Stack을 사용한다.(stack overflow주의)
- 대부분의 재귀는 반복문과 스택 구조로 변환할 수 있다.

## 예시
- 팩토리얼
- 트리
- DFS
- 분할 정복 알고리즘

## 면접 대비 질문

### 재귀함수에 대해 설명해 주세요.
재귀 함수는 함수가 자기 자신을 호출하여 문제를 해결하는 방식의 함수이다.  
주로 큰 문제를 동일한 구조의 더 작은 문제로 나누어 해결할 수 있을 때 사용된다.  
재귀 함수는 반드시 재귀 호출과 종료 조건(Base Case) 을 포함해야 하며, 종료 조건이 없으면 무한 호출이 발생하여 Stack Overflow가 발생할 수 있다.  
대표적인 예로는 트리 탐색, DFS, 분할 정복 알고리즘 등이 있다. 


### 재귀 함수의 동작 과정을 Call Stack을 활용해서 설명해 주세요.
귀 함수가 호출될 때마다 함수의 실행 정보가 Call Stack에 차례대로 쌓인다.  
재귀 호출이 계속 진행되다가 종료 조건(Base Case)에 도달하면 더 이상 재귀 호출이 이루어지지 않고, 스택에 쌓여 있던 함수들이 역순으로 반환되면서 계산이 진행된다.  
즉 재귀 함수는 Call Stack을 이용하여 LIFO 방식으로 실행된다.

### 언어의 스펙에 따라, 재귀함수의 최적화를 진행해주는 경우가 있습니다. 어떤 경우에 재귀함수의 최적화가 가능하며, 이를 어떻게 최적화 할 수 있을지 설명해 주세요.
일부언어에서는 Tail Recursion Optimization(TCO)을 통해 재귀 호출을 최적화할수있다.  
Tail Recursion조건은 재귀호출이 함수의 마지막 연산이어야한다.  
재귀호출 이후 추가 연산이 없고, 컴파일러가 반복문으로 변환이 가능하다는 특징이 있다.  
최적화 방식으로 컴파일러는 call stack push 대신 현재 스택 프레임 재사용을 수행한다.  
재귀호출을 반복문 형태로 변환이 가능해져서 stack overflow위험이 줄어든다.  
하지만, 자바는 tail Recurion Optimization을 지원하지않는다.  
자바에서는 반복문으로 변경하거나, 명시적인 stack을 사용하여 구현하는것이 안전하다.
