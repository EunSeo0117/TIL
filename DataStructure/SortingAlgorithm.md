# 정렬 알고리즘

## 개념
데이터를 특정 기준에 따라 일정한 순서로 재배열하는 알고리즘이다.  
숫자나 문자열을 오름차순 또는 내림차순으로 정렬하는데 사용된다.  

## 특징
- 비교기반 정렬 : 두값을 비교하여 정렬 수행
- 안정 정렬 : 같은 값의 상대적 순서가 유지됨
- 불안정 정렬 : 같은 값의 순서가 바뀔수있음
- 내부 정렬 : 메모리 내부에서 수행
- 외부 정렬 : 데이터가 너무 커서 디스크 사용

## 종류
- 단순 정렬(구현이 단순하고 작은데이터에 적합)
  - 버블정렬
  - 선택정렬
  - 삽입정렬
- 분할 정복 기반 정렬(큰 데이터 처리에 효율적, Divide Conquer방식)
  - 병합정렬
  - 퀵정렬
- 특수 정렬(특정 데이터 조건에서 매우 빠름, 비교기반이 아닐수있음)
  - 힙정렬
  - 카운팅 정렬
  - 래딕스 정렬

## 시간 복잡도
| 알고리즘           | 평균         | 최악         | 특징             |
| -------------- | ---------- | ---------- | -------------- |
| Bubble Sort    | O(n²)      | O(n²)      | 구현 단순          |
| Selection Sort | O(n²)      | O(n²)      | 교환 횟수 적음       |
| Insertion Sort | O(n²)      | O(n²)      | 거의 정렬된 데이터에 빠름 |
| Merge Sort     | O(n log n) | O(n log n) | 안정 정렬          |
| Quick Sort     | O(n log n) | O(n²)      | 평균적으로 가장 빠름    |
| Heap Sort      | O(n log n) | O(n log n) | 추가 메모리 적음      |
| Counting Sort  | O(n + k)   | O(n + k)   | 정수 범위 제한       |
| Radix Sort     | O(nk)      | O(nk)      | 자리수 기반 정렬      |

## 사용사례
- 검색속도 향상
- 데이터 분석
- 데이터 처리 및 시각화
- 다른 알고리즘의 전처리단계

## 면접대비 질문

### 정렬 알고리즘에 대해 설명해 주세요.
정렬알고리즘은 데이터를 특정 기준에 따라 일정한 순서로 재배열 하는 알고리즘이다.  
보통 숫자난 문자열을 오름차순 또는 내림차순으로 정렬한다.  
정렬은 검색성능향상, 데이터분석 및 처리, 다른 알고리즘 전처리단계 등에서 사용된다.  
정렬은 비교기반 정렬, 비교기반이 아닌 정렬 두가지로 나눌수있고, 비교기반 정렬의 이론적인 시간복잡도 하한은 n log n이다.

### Quick Sort와 Merge Sort를 비교해 주세요.
두 정렬 모두 디바이드&퀀커 방식을 사용하는 알고리즘이다.  
퀵정렬은 피벗을 기준으로 배열을 두분류로 나누고 재귀적으로 정렬하는 방식이다.  
시간복잡도는 n log n인데, 피벗 선택이 좋지않으면 최대 n^2까지 성능이 떨어질수있다.  
대신 추가 메모리를 거의 사용하지않는 인플레이스 정렬이다. 
반대로 병합정렬은 배열을 반으로 나눈뒤 정렬된 배열을 병합하는 방식이다.  
항상 n log n의 시간복잡도를 보장하지만, 병합과정에서 o(n)의 추가 메모리가 필요하다.  
또한 상태기반 정렬이다. 

### Quick Sort에서 O(N^2)이 걸리는 예시를 들고, 이를 개선할 수 있는 방법에 대해 설명해 주세요.
이미 정렬된 배열에서 피벗을 첫번쨰 원소로 선택하는 경우 최악의 성능이 발생한다.  
분할이 매우 불균형하게 이뤄지고 이과정이 반복되면 최대 n^2의 성능이 된다.  
이를 개선하기위해 랜덤피벗, 인트로정렬 등을 사용할수있다.  

### Stable Sort가 무엇이고, 어떤 정렬 알고리즘이 Stable 한지 설명해 주세요.
상같은 값을 가진 데이터의 상대적인 순서를 유지하는 정렬 알고리즘이다.  
병합정렬, 삽입정렬, 버블정렬, 카운팅정렬, 래딕스정렬등이 있다.
퀵정렬이나 힙정렬은 unstable정렬이다.

### Merge Sort를 재귀를 사용하지 않고 구현할 수 있을까요?
바텀업 병합정렬로 구현이 가능하다.  
일반적인 병합정렬은 배열을 재귀적으로 분할하지만, 바텀업 방식은 크기가 1인 배열부터 시작해서  
점점 병헙 범위를 늘려가는 방식이다.  
시간복잡도는 동일하게 n log n이다.

### Radix Sort에 대해 설명해 주세요.
숫자의 자리수를 기준으로 정렬하는 알고리즘이다.  
여러 숫자가 있을때 1의 자리 기준 정렬, 10의 자리 기준 정렬, 100의자리 기준 정렬 순서로 정렬한다.  
각 자리 정렬 단계에서는 stable 정렬, 보통은 카운팅 정렬을 사용한다. 
래딕스 정렬의 시간복잡도는 O(nk)이다. k는 숫자의 자리수이다.  
비교기반 정렬이 아니기때문에 특정조건에서는 n log n보다 빠르게 동작할수있다. 

### Bubble, Selection, Insertion Sort의 속도를 비교해 주세요.
세 알고리즘의 평균 및 최악의 시간복잡도 모두 n^2이다.
하지만, 실제 성능의 차이는 존재한다.  
삽입정렬은 펴균적으로 가장 빠르다, 선택정렬은 항상 동일한 비교횟수를 가지고, 버블정렬은 교환 연산이 많아 가장느리다.  
실제로 삽입정렬이 가장 많이 사용되는 알고리즘이다.

### 값이 거의 정렬되어 있거나, 아예 정렬되어 있다면, 위 세 알고리즘의 성능 비교 결과는 달라질까요?
데이터가 이미 정렬되어있다면 삽입정렬은 매우 효율적이다. 이 경우 삽입정렬의 시간복잡도는 o(n)까지 줄어든다.  
반면 버블정룔과 선택정렬은 여전히 n^2이기때문에 성능 차이가 크게 발생한다. 

### 주요 언어(Java, Python, JavaScript 등)에서는 어떤 정렬 알고리즘을 사용하여 정렬 함수를 제공하고 있을까요?
자바에서는 primitive type정렬에는 dual pivot quick정렬을 사용하고 object정렬은 timsort를 사용한다.
파이썬은 Timsort를 사용한다.  
cpp의 경우 intro정렬을 사용한다. IntroSort는 Quick Sort, Heap Sort, Insertion Sort를 결합한 알고리즘이다.

### 정렬해야 하는 데이터는 50G 인데, 메모리가 4G라면, 어떤 방식으로 정렬을 진행할 수 있을까요?
이 경우 external sort, 특히 external merge sort를 사용한다.  
먼저 데이터를 메모리에 들어갈수있는 크기로 나누어 각각을 메모리에서 정렬한다.
그후 정렬된 여러개의 파일을 머지 방식으로 병합하여 최종 정렬 결과를 생성한다.  
디스크 기반 정렬방식이며, 대용량 데이터 처리 시스템에서도 일반적으로 사용되는 방식이다.
