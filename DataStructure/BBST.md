# BBST

## 개념
균형 이진 탐색 트리는 이진탐색트리의 성질을 유지하면서,  
트리의 높이가 한쪽으로 치우치지 않도록 균형을 유지하는 자료구조이다.  
일반 이진탐색트리는 삽입 순서에 따라 한쪽으로 치우쳐 최악의 경우 O(n)이 될수있지만,  
균형이진탐색트리는 트리높이를 logN으로 유지하며 탐색, 삽입, 삭제를 안정적으로 수행한다.

## 특징
- 이진탐색트리의 기본성질을 유지해서 왼쪽서브트리 < 부모 < 오른쪽서브트리 구조를 유지한다
- 트리높이가 log2N수준으로 유지되고, 최악의 경우에도 선형구조로 붕괴되지않는다.
- 삽입/삭제후 균형이 꺠지면 오른쪽회전, 왼쪽회전을 수행하여 균형을 맞춘다

## 시간복잡도
- 탐색: log N
- 삽입: log N
- 삭제: log N

## 대표 종류
- AVL Tree
- Red-Black Tree
- B-Tree(디스크 기반 구조에서 사용)
- 2-3 Tree


## 면접 질문 대비

### BBST (Balanced Binary Search Tree) 와, 그 종류에 대해 설명해 주세요.
균형이진탐색 트리로 이진탐색트리의 성질을 유지하면서 트리의 높이가 한쪽으로 치우지지않도록 균형을 유지하는 자료구조이다.  
일반 이진탐색트리는 삽입순서에 따라 한쪽으로 치우쳐 최악의 경ㅇ우 시간복잡도가 o(n)까지 증가할수있다.  
균형이진탐색트리는 균형을 유지하여 탐색, 삽입, 삭제 연산을 모두 log N으로 보장한다.  
대표적인 종류로는 AVL Tree, Red-Black Tree, 2-3Tree, 2-3-4 Tree, B-Tree가 있다. 


### Red Black Tree는 어떻게 균형을 유지할 수 있을까요?
노드를 Red또는 Black색으로 구분하고, 특정규칙을 강제하여 트리높이를 제한한다.  
삽입이나 삭제로 인해 규칙이 꺠지면, 1.색변경, 2.회전을 통해 다시 규칙을 만족하도록 복구한다.  
모든 경로에서 Black노드의 갯수를 동일하게 유지하여 트리높이가 2*log N이하로 제한되도록 만든것이다.  
완전한 균형은 아니지만, 느슨한 균형을 유지하여 성능을 보장한다. 

### Red Black Tree의 주요 성질 4가지에 대해 설명해 주세요.
1. 모든 노드는 Red, Black이다.
2. 루트노드는 항상 Black이다. 
3. Red노드의 자식은 모두 Black이다(Red-Red연속금지)
4. 어떤 노드에서 리프까지 가는 모든 경로는 동일한 갯수의 Black노드를 가진다.
이 성질 덕분에 트리 높이가 최대 2*log N으로 제한된다.


### 2-3-4 Tree, AVL Tree 등의 다른 BBST 가 있음에도, 왜 Red Black Tree가 많이 사용될까요?
성능과 구현 복잡도의 균형때문이다. 
Red-Black Tree는 완전 균형은 아니지만, 회전 비용을 줄이면서도   
O(log N)을 안정적으로 보장하는 실용적인 균형 트리이기 때문에  실무에서 가장 널리 사용된다.  

AVL Tree보다 회전이 적다. AVL은 매우 엄격한 균형을 유지하고, 삽입/삭제시 회전이 더 자주발생한다.  
Red-Black은 상대적으로 회전이 적어 실제 환경에서 더 빠른경우가 많다.  
또한, 삽입/삭제가 더 단순하다. 2-3-4 Tree는 개념적으로 좋지만, 구현이 복잡하다.  
Red-Black은 이진트리 형태를 유지하므로 구현이 비교적 직관적이다.   
마지막으로, 실무표준으로 자리잡고있다. 자바 TreeMap, TreeSet, Cpp의 std::map, std::set이있다.
