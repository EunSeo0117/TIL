# Graph

## 개념
그래프는 정점과 간선으로 이루어진 자료구조이다.  
정점은 객체를 의미하고, 간선은 정점간의 관계를 의미한다.  
그래프는 객체 간의 연결 관계를 표현할 때 사용되는 비선형 자료구조이다.  

## 특징
그래프는 G = (V,E) | V:정점집합, E:간선집합 으로 표현한다.  

1. 비선형 자료구조이다.
2. 순환이 존재할수있다. (a->b->c->a)
3. 방향성 존재 여부(방향그래프/무방향그래프)가 있다.
4. 가중치 존재여부(가중치 그래프/비가중치그래프)가 있다.
5. 희소그래프 / 밀집그래프가 있다(Sparse그래프 간선적음, Dense 그래프 간선 많음)

## 예시
- 네트워크 구조(컴퓨터네트워크)
- 지도 및 경로 탐색
- SNS 관계
- 의존성 도구(빌드시스템)
- 추천 시스템

## 종류
1. 방향그래프
   - 간선에 방향이 존재한다.
   - a->b는 가능하나 b->a는 불가능하다.
2. 무방향 그래프
   - 간선에 방향이 없다
   - a<->b 양방향 연결 의미한다.
3. 가중치 그래프
   - 간선에 비용이 존재한다.
   - a->b(cost=5)
   - ex) 지도경로, 네트워크 레이턴시
4. 비가중 그래프
   - 간선에 비용이 없다.
   - 모든 간선의 비용을 1로 간주한다.
5. DAG(Directed Acyclic Graph)
   - 방향그래프이며 사이클이 존재하지 않는 그래프
   - 컴파일 의존성, 작업스케쥴링

## 시간복잡도
1. 인접행렬 : 정점이 n개일떄, n*n배열을 사용한다.

| 연산              | 시간복잡도 |
| --------------- | ----- |
| 간선 존재 확인        | O(1)  |
| 정점의 모든 인접 정점 탐색 | O(N)  |
| 공간복잡도           | O(N²) |

2. 인접 리스트 : 각 정점마다 연결된 정점을 리스트로 저장한다.

| 연산              | 시간복잡도     |
| --------------- | --------- |
| 간선 존재 확인        | O(degree) |
| 정점의 모든 인접 정점 탐색 | O(degree) |
| 공간복잡도           | O(V + E)  |


## 면접대비 질문

### 그래프 자료구조에 대해 설명하고, 이를 구현할 수 있는 두 방법에 대해 설명해 주세요.
그래프는 정점과 간선으로 이루어진 자료구조로, 객체간의 관계를 표현할때 사용된다. 그래프를 구현하는 대표적인 방법은 두가지가 있다.  
1. 인접행렬 2. 인접리스트
인접행렬은 간선 확인이 빠르지만 공간 사용이 많고, 인접리스트는 공간 효율이 좋지만 간선 확인이 상대적으로 느리다. 

### 각 방법에 대해, "두 정점이 연결되었는지" 확인하는 시간복잡도와 "한 정점에 연결된 모든 정점을 찾는" 시간복잡도, 그리고 공간복잡도를 비교해 주세요.
| 구현 방식  | 연결 확인     | 인접 정점 탐색  | 공간     |
| ------ | --------- | --------- | ------ |
| 인접 행렬  | O(1)      | O(N)      | O(N²)  |
| 인접 리스트 | O(degree) | O(degree) | O(V+E) |
degree = 해당 정점의 연결 간선수

### 정점의 개수가 N개, 간선의 개수가 N^3 개라면, 어떤 방식으로 구현하는 것이 효율적일까요?
그래프가 매우밀집이다.(Dense Graph)
간선수가 매우 많아서 인접행렬 방식이 더효율적이다.  
- 인접리스트 -> O(V+E) = O(N^3)
- 인접행렬 -> O(N^2) 

### 사이클이 없는 그래프는 모두 트리인가요? 그렇지 않다면, 예시를 들어주세요.
모두 트리가 아니다. 트리는 1.사이클이 없음 2. 연결그래프 3.간선수 =  n-1이어야한다. 사이클이 없지만, 연결되지 않은 그래프는 트리가 아니다. 

### 그래프에서, 최단거리를 구하는 방법에 대해 설명해 주세요.
| 알고리즘           | 특징           |
| -------------- | ------------ |
| BFS            | 비가중 그래프 최단거리 |
| Dijkstra       | 양수 가중치       |
| Bellman-Ford   | 음수 간선 허용     |
| Floyd-Warshall | 모든 정점 쌍 최단거리 |

### 트리에서는 어떤 방식으로 최단거리를 구할 수 있을까요? (위 방법을 사용하지 않고)
트리는 두 정점 사이의 경로가 항상 하나다. 따라서 dfs,bfs를 이용해 탐색하면 해당경록 곧 최단경로다. 

### 다익스트라 알고리즘에서, 힙을 사용하지 않고 구현한다면 시간복잡도가 어떻게 변화할까요?
힙 사용시 O(v+e) log V인데, 힙사용을 하지 않고 배열로 최소 거리 정점을 찾으면 매번 O(V)이다. 그걸 총 v번반복하므로 O(V^2)이다.

### 정점의 개수가 N개, 간선의 개수가 N^3 개라면, 어떤 알고리즘이 효율적일까요?
이 경우 그래프가 매우 Dense Graph이다.  
플로이드-워셜 알고리즘 시간복잡도가 O(V^3)이며,  
간선수와 유사한 수준이기 때문에, 플로이드-워셜이 적합하다.

### A* 알고리즘에 대해 설명해 주세요. 이 알고리즘은 다익스트라와 비교해서 어떤 성능을 낼까요?
A*알고리즘은 휴리스틱 기반 최단경로 탐색 알고리즘입니다.  
다익스트라는 f(n) = g(n)
A* 는 f(n) = g(n) + h(n)
- g(n) : 시작점 -> 현재노드 비용
- h(n) : 현재노드 -> 목표까지 예상비용
즉 목표 방향으로 탐색을 유도한다. 
성능은 다익스트라는 그래프를 전체 탐색 가능하고, A*는 목표방향만 탐색한다.  
따라서 일반적으로 A*가 훨씬 빠르다. 
대표적으로 게임경로 탐색, 지도서비스가있다.

### 음수 간선이 있을 때와, 음수 사이클이 있을 때 각각 어떤 최단거리 알고리즘을 사용해야 하는지 설명해 주세요.
| 상황        | 알고리즘         |
| --------- | ------------ |
| 음수 간선 존재  | Bellman-Ford |
| 음수 사이클 존재 | 최단거리 정의 불가능  |
음수 사이클이 존재하면 같은 경로를 계속 돌면서 비용을 무한히 줄일수잇기에 최단거리가 정의되지 않는다.  
벨만포드 알고리즘은 음수 사이클을 탐지할수있다.
